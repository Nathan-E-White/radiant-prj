# Separate Resident Source, Simulation Telemetry, Simulation Result, and Twin State Streams

| Field | Value |
| --- | --- |
| Document ID | ADR-0005 |
| Revision | 1.0 |
| Status | Accepted |
| Owner | Software |
| Baseline | Simulator Workbench backend dataflow slice |

## Status

Accepted

## Context

Simulator Workbench needs to show measured, simulated, and imputed values without collapsing them into a generic metric stream. Resident SCADA stand-ins are long-running public-safe measured sources. SimOps workers also produce operational telemetry, but operational telemetry is not engineering result state. Synthetic engineering or physics-like results are separate run-scoped simulated result frames. Imputed state is produced only after the digital twin projector consumes measured inputs, simulated result state, model identity, and artifact lineage.

## Decision

Use four streams with separate contracts and projections:

- `scada.telemetry.v1` carries resident measured frames with `valueBasis=measured`.
- `simops.telemetry.v1` carries run-scoped operational worker telemetry.
- `simops.results.v1` carries synthetic simulated result frames with `valueBasis=simulated`.
- `digital-twin.state.v1` carries current twin state values, including `valueBasis=imputed` values emitted only by the twin projector.

The internal gateway owns token-gated ingest for SCADA sources, SCADA telemetry, SimOps telemetry, and SimOps results. Projection writers materialize Redpanda coordinates into Postgres read models and Iceberg tables. The twin projector records lineage from measured tags, simulation run/result, model id, and artifact ids before publishing twin state.

## Consequences

Measured, simulated, and imputed values stay traceable through transport, storage, and read APIs. The frontend follow-up can query read-only Workbench APIs without learning broker, database, or lake credentials. The cost is a more complicated local compose graph, so `bun run simulator-workbench:dataflow:smoke` is the objective evidence check for the full backend path.
