# Add Phaser Board Rendering For Fleet Board Only

| Field | Value |
| --- | --- |
| Document ID | ADR-0006 |
| Revision | 0.2 |
| Status | Accepted |
| Owner | Software |
| Baseline | Fleet Board feature branch |

## Status

Accepted

## Context

The presentational Simulator Workbench slice originally deferred Phaser, PixiJS, and Three.js because the first need was inspectable value-basis presentation, not a high-frequency game surface. Fleet Board changes that premise. The new surface is a playable logistics board inside Simulator Workbench: grid-snapped facility cards, animated service routes, and roaming Inspector and Trouble pawns.

React remains the right owner for workbench state, measured/imputed/simulated panels, lineage, commercial display basis, accessibility, and app layout. Phaser is useful for the board camera, drag/drop feel, route animation, and pawn motion.

## Decision

Use Phaser only for the Fleet Board canvas scene. React owns the game reducer, Workbench projection adapter, selected state, truth/readout panels, layout, and browser accessibility.

The Phaser scene receives a small scene model derived from:

- the pure Fleet Board game state;
- the existing Simulator Workbench projection;
- the Workbench-derived gameplay modifiers.

Phaser does not fetch backend data, own Value Basis semantics, hold durable game state, render dense tables, or replace the Simulator Workbench panels.

## Consequences

Fleet Board can feel like a game without turning the workbench into a fake control console. The cost is a new frontend dependency and a second rendering lifecycle. That cost is controlled by keeping the reducer pure, keeping Phaser behind a React component, and covering the canvas mount and basic interaction with Playwright.

The first Fleet Board slice remains projection-backed and local. ADR-0007 now governs the separate backend-backed horizon. Its mediated game-intent, identity, persistence, security, recovery, and live-read boundaries revise this decision's earlier local-only consequence without giving Phaser backend access or state ownership.
