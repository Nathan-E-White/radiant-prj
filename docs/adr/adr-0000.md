# Choosing Phaser (PixiJS/WebGL) for Accelerated Browser-Side Telemetry Pipeline Visualization

## Status
Accepted

## Context
The SimEngine Operator Console requires high-frequency rendering of spatial kinetics, core fluid dynamics, and a 16-drum reactor P&ID loop running at a 20Hz data ingestion rate. Traditional HTML canvas methods or browser-heavy DOM/SVG rendering elements create severe performance issues when processing dense, high-frequency telemetry updates.

Frequent layout recalculations choke the browser's single-threaded event loop. This leads to user interface lagging, thread blocking, and an unacceptably high local CPU overhead. To maintain real-time safety tracking operations within our critical < 2.0ms Hardware-in-the-Loop (HIL) latency window, the interface graphics layer must remain completely non-blocking.

## Decision
We will use Phaser (built on the PixiJS WebGL graphics engine) as our core 2D rendering framework inside the client browser.

The interactive P&ID loop and core thermal mesh graphics will run entirely inside a hardware-accelerated Phaser canvas block. This canvas block sits directly within a clean, static structural frame handled by CSS. All heavy mathematical visuals, layout rotations, and color-mapping changes will be offloaded onto the local GPU graphics pipelines via WebGL shaders, locking the interface rendering rate at a steady 60 frames per second.

```text
[WebSocket Ingestion Node Thread]
               |
               v (JSON Packet Unpack)
  [Global Shared Data Registers]
               |
               +-----------------------+-----------------------+

                                       |                       |
                                       v (Static Text Fields)  v (WebGL Memory Context)
                           [Tailwind CSS Grid Layout]   [Embedded Phaser Scene Canvas]
                           • Slurm Text Labels          • 16-Drum Rotational Vector Math
                           • Buffer Metrics Tables      • Fluid Path Flow Particle Traces
```

## Consequences

### Positive

#### GPU Offloading
Offloads heavy mathematical drawings from the CPU main thread to local hardware-accelerated WebGL pipelines. This maintains a sub-millisecond layout thread profile.

#### Fixed Frame Lifecycle
Employs a reliable preload -> create -> update loop model. The UI responds smoothly to user inputs even when processing high-volume transient failure spikes.

#### Low Memory Overhead
Bypasses heavy frontend single-page application framework layers (such as React's virtual DOM reconciliation loop), reducing memory allocation footprint.

### Negative

#### Dependency Additions
Adds a client-side game engine dependency to an industrial engineering dashboard, which requires explicit framing to team reviewers during deployment.

#### Form Control Integration
Phaser handles dense alphanumeric input forms poorly. We must manage user button inputs and text blocks outside Phaser using native HTML overlay forms.
