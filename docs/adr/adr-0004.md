## Context
A panel to act as a low-level debugging interface for the HPC cluster systems engineer, visualizing fabric loop latency, link saturation, and MPI non-blocking collective communication overhead across the microreactor's unit cells. Gathering network packet drop logs, port errors, or fabric link temperature changes directly from hardware counters (such as InfiniBand or HPE Slingshot controllers) inside the high-frequency network loop introduces fatal polling delays. Accessing local bus adapters synchronously would block the core data loop, driving up inter-node synchronization latency and destabilizing the real-time HIL tracking interface.

We will decouple hardware diagnostic collection from the primary network loop by building an Asynchronous Shared Memory Segmentation framework.

A dedicated low-priority system background process will query the physical fabric ports and hardware counter buffers at a slower, decoupled rate (2Hz / every 500ms). This driver writes raw link utilization and port metrics into a shared kernel memory segment. The primary Go telemetry broker then reads from this pre-populated memory map during its fast 20Hz loop, passing the metrics upstream inside the global JSON frame payload without stalling to query hardware devices.

```text
[Physical InfiniBand/Slingshot Fabric Fabric]
                    |
                    v (Slow 2Hz Query Thread)
        [Low-Level Hardware Collector]
                    |
                    v (Zero-Copy Write)
       [Shared Kernel Memory Segment] <--- [Go 20Hz Telemetry Main Loop] ---> [WebSocket Client]
```

## Consequences
### Positive
#### Zero Polling Bloat
The primary data pipeline loop performs a zero-copy read straight out of local RAM, avoiding physical system bus queries and maintaining low execution times.
#### Network Isolation
If an internal switch link fails completely or hangs during a query, the hardware collection process will block safely in the background without bringing down the client's live reactor dashboard interface.
### Negative
#### Telemetry Staleness (Phase Shift)
Fabric topology metrics lag by up to 500ms behind the real-time core physics data, which is acceptable since network routing changes happen on a macroscopic timescale compared to microsecond fuel transients.
