# Separate Control-Plane, Event-Fanout, And Analytical Storage Authority

## Status

Accepted

## Context

ADR-0005 separates Measured State, operational telemetry, Simulated Result State, Twin State, and Lineage by meaning. The implemented dataflow also makes a separate architectural choice about where each kind of operational fact is authoritative: API and Run control state, ordered fanout, projections, and analytical artifacts have different recovery and consistency needs.

Treating Postgres, Redpanda, and Iceberg as interchangeable copies would make retry, replay, snapshot consistency, and incident recovery ambiguous.

## Decision

Postgres is authoritative for control-plane records, idempotency, current operational projections, protected configuration, and the transactional state required to recover them. Redpanda is the ordered durable fanout source for accepted telemetry, result, state, and lineage events; topic, partition, and offset are retained where a projection must establish provenance or replay position. Iceberg is the durable analytical-artifact store and is populated only through append/readback semantics that make an accepted artifact observable.

Projection writers may rebuild materialized state from their declared event inputs, but no projection or Iceberg artifact supersedes the control-plane authority for Run lifecycle, request idempotency, or protected configuration. A coherent Workbench Snapshot is published only from one accepted generation; browser callers use read-only projections and receive no broker, database, lake, object-store, or runtime credentials.

## Consequences

The dataflow has explicit recovery ownership: control-plane recovery is transactional, event recovery is coordinate-aware, and analytical recovery verifies committed artifacts. This adds cross-store discipline but prevents a convenient replica from quietly becoming an undefined source of truth. The semantic stream rules remain in ADR-0005; the active dataflow is recorded in the [Simulator Workbench Backend Dataflow Slice](../design/simulator-workbench-backend-dataflow-slice.md).
